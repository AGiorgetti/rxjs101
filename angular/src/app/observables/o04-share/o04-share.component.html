Stream with an initial value (not shared)

notShared$ is NOT SHARED, this means that for every async pipe, the functionality to calculate the value will be executed again

  • First = {{notShared$|async}}
  • Second = {{notShared$|async}}

Both expressions are displayed correctly

Stream with an initial value (shared)

shared$ is SHARED, this means that for all async pipes, the functionality will be executed only once

  • First = {{shared$|async}}
  • Second = {{shared$|async}}

Only the first expression is displayed correctly

The share() operator will publish the first value on the first subscription. The first async pipe will subscribe and get the value. The second async pipe however will subscribe after that value has already been emitted and therefore will miss it.

Stream with an initial value (shareReplay)

sharedReplay$ is SHARED, this means that for all async pipes, the functionality will be executed only once

  • First = {{sharedReplay$|async}}
  • Second = {{sharedReplay$|async}}

Both expressions are displayed correctly and the functionality is only executed once

the shareReplay(1) operator will do a publishReplay(1).refcount() behind the scenes which keeps track of the last element in the stream, and will emit it again when the second subscription is made.
